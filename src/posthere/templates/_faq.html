FAQ

What the heck is this thing?

POSThere.io is a simple service for developers to help debug API calls and web hooks.

When your code POSTs to someone else's code (using their web API, or if you're acting as a web hook) it can be hard to iterate, debug and validate your code without the burden of mocking your POSTs or temporarily servicing your own POST requests.

Wouldn’t it be nice to be able to instantly see exactly what your code POSTed at any time in as easy, clear browser interface? Now you can with POSThere.io. After POSTing your JSON, XML, or form fields, point your web browser at the same URL and you’ll see a clear history and details of all the POST requests you made. Neat, huh?

Another great use for POSThere.io is when you're using someone else's web hook for the first time. Point it at POSThere.io first so you can see exactly what will be POSTed, then you can write your own code to process the request.

How does POSThere.io work?

Simply decide if you want to use HTTP or HTTPs, then come up with a string that makes sense for the POST you’re attempting, append it to posthere.io, and POST away.

http://posthere.io/test-my-twitter-code

https://posthere.io/a-webhook-callback-test

https://posthere.io/another/test/with/complexity?api=123

What kind of data can I POST?

You can POST, PUT or PATCH any kind of data up to 1 megabyte in length, but POSThere.io is optimized for textual content, and will even do fancy syntax highlighting when you send JSON, XML or form encoded data.

Can I PUT to POSThere.io?

Yep.

Can I PATCH to POSThere.io?

Indeed.

Can I change the HTTP status that’s returned when I POST?

Yes. Just add ?status=xxx to the end of your request, such as ?status=404.

How long will my requests stay visible?

Your requests are available for up to 24 hours. In case we are super busy and run out of storage space, we might remove your older requests earlier than that.

How many requests will stay available on POSThere.io?

Up to your last 100 requests per unique URL. Past that we start to recycle the bits of the older ones.

Can other people see my POSTed data?

Yes they can, if you share the URL with them or they guess it. POSThere.io intentionally uses security by obscurity to keep things super simple. Don't POST any sensitive data to POSThere.io.

I don't trust you with my test data. It's very secret and sensitive and gets its feelings hurt easily. What do you have to say about that?

We thought of that. We made POSThere.io an open source project. Check the code and verify we don’t do anything nefarious, or host it yourself internally.

Go grab a copy from the [Github repository](https://github.com/path/posthere.io) and check out the code or run your own private instance. We don't mind, we promise.

Oops! I didn't mean to POST that. Can I get rid of it?

No! It's ours forever!

Not really. Just click the delete link on the result page for the URL in your browser, or hit the URL you posted to with a DELETE request from your programming language or cURL. It's all good.

Can I get the results of my POST requests with an API?

Why, of course you can. Just change your POST/PUT or PATCH to a GET and access the same URL. Make sure you include an 'Accept' header of 'application/json'. You'll get back a JSON array of all your requests. The contents of which should be fairly self-explanatory.

Can I use POSThere.io from JavaScript in the browser?

What do we look like? Amateurs? We got this puppy all [CORS](http://en.wikipedia.org/wiki/Cross-origin_resource_sharing)'d up. You can POST/PUT/PATCH to your heart's content from JavaScript in the browser.

Are there any URLs I can’t POST to?

The URL name space of http://posthere.io/_/... is reserved for the POSThere.io website. You also can't use http://posthere.io/index. Any other URL should be fair game... POST away!

How come POSThere.io doesn't butter my bread in the morning and tuck me in at night?

I don't know. Maybe it should. If you need more features then you should fork the [repo on GitHub](https://github.com/path/posthere.io) and add in a bread buttering feature. Use it on your own private instance and enjoy, or send us a pull request and we promise to look at it.

If you're not quite up to building the tuck me into bed feature yourself, you can request it with the Feedback widget at the bottom of the page. If it's a brilliant feature idea, we just might build it for you.
